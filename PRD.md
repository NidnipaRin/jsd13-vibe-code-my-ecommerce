# Product Requirements Document (PRD)

## 1. ภาพรวมผลิตภัณฑ์

สร้างเว็บแอปพลิเคชันอีคอมเมิร์ซแบบ Full-stack ด้วย MERN Stack เพื่อให้ลูกค้าค้นหาและสั่งซื้อสินค้าออนไลน์ได้ และให้ผู้ดูแลระบบจัดการสินค้าและคำสั่งซื้อได้จากระบบเดียว

> ที่มา: เอกสาร `apps/README.md` ระบุเป้าหมายว่าเป็น “MERN stack e-commerce web app” รายละเอียดใน PRD นี้จึงเป็นขอบเขต MVP ที่เสนอเพื่อใช้เริ่มพัฒนา และสามารถปรับได้ตามความต้องการธุรกิจ

## 2. เป้าหมาย

- มอบประสบการณ์เลือกซื้อสินค้าออนไลน์ที่ใช้งานง่ายบนเดสก์ท็อปและมือถือ
- รองรับกระบวนการซื้อหลักตั้งแต่ค้นหาสินค้า เพิ่มลงตะกร้า จนถึงส่งคำสั่งซื้อ
- ให้ผู้ดูแลระบบจัดการแคตตาล็อกสินค้าและติดตามคำสั่งซื้อได้
- วางโครงสร้างระบบให้ขยายต่อเป็นเว็บและแอปมือถือได้

## 3. ผู้ใช้เป้าหมาย

| บทบาท | ความต้องการหลัก |
| --- | --- |
| ผู้เยี่ยมชม | ดู ค้นหา และกรองสินค้าได้โดยไม่ต้องเข้าสู่ระบบ |
| ลูกค้า | สมัครสมาชิก จัดการตะกร้า สั่งซื้อ และดูประวัติคำสั่งซื้อ |
| ผู้ดูแลระบบ | จัดการสินค้า หมวดหมู่ สต็อก และสถานะคำสั่งซื้อ |

## 4. ขอบเขต MVP

### 4.1 ฝั่งลูกค้า

- หน้าหลักแสดงสินค้าเด่น หมวดหมู่ และโปรโมชั่น (ถ้ามี)
- หน้ารวมสินค้า พร้อมค้นหา กรองตามหมวดหมู่/ราคา และเรียงลำดับสินค้า
- หน้ารายละเอียดสินค้า: รูปภาพ ราคา รายละเอียด สต็อก และปุ่มเพิ่มลงตะกร้า
- ตะกร้าสินค้า: เพิ่ม ลด ลบรายการ และคำนวณยอดรวม
- สมัครสมาชิก เข้าสู่ระบบ และออกจากระบบ
- Checkout: ระบุข้อมูลจัดส่ง ตรวจสอบรายการ และยืนยันคำสั่งซื้อ
- หน้าประวัติและรายละเอียดคำสั่งซื้อของผู้ใช้

### 4.2 ฝั่งผู้ดูแลระบบ

- เข้าถึงหน้าจัดการที่จำกัดเฉพาะบัญชีผู้ดูแลระบบ
- สร้าง แก้ไข ลบ และดูรายการสินค้า
- จัดการข้อมูลสินค้า ได้แก่ ชื่อ ราคา หมวดหมู่ คำอธิบาย รูปภาพ และจำนวนคงเหลือ
- ดูรายการคำสั่งซื้อและปรับสถานะ เช่น `pending`, `paid`, `processing`, `shipped`, `completed`, `cancelled`
- ดูข้อมูลลูกค้าที่เกี่ยวข้องกับคำสั่งซื้อเท่าที่จำเป็นต่อการจัดส่งและบริการลูกค้า

## 5. ข้อกำหนดเชิงหน้าที่

| รหัส | ข้อกำหนด |
| --- | --- |
| FR-01 | ระบบต้องให้ผู้ใช้สมัครสมาชิกและเข้าสู่ระบบด้วยอีเมลและรหัสผ่านได้ |
| FR-02 | ระบบต้องตรวจสอบสิทธิ์ผู้ใช้และจำกัดการใช้งานส่วนผู้ดูแลระบบ |
| FR-03 | ผู้ใช้ต้องค้นหา กรอง และเรียงลำดับสินค้าได้ |
| FR-04 | ผู้ใช้ต้องจัดการจำนวนสินค้าในตะกร้าได้ โดยจำนวนต้องไม่เกินสต็อก |
| FR-05 | ระบบต้องเก็บข้อมูลคำสั่งซื้อ พร้อมรายการสินค้า ยอดรวม ที่อยู่จัดส่ง และสถานะ |
| FR-06 | ลูกค้าต้องดูคำสั่งซื้อของตนเองได้เท่านั้น |
| FR-07 | ผู้ดูแลระบบต้องจัดการสินค้าและสถานะคำสั่งซื้อได้ |
| FR-08 | ระบบต้องลดหรือจองสต็อกตามกติกาที่ทีมกำหนดเมื่อมีการยืนยันคำสั่งซื้อ |

## 6. ข้อกำหนดที่ไม่ใช่เชิงหน้าที่

- Responsive รองรับหน้าจอมือถือ แท็บเล็ต และเดสก์ท็อป
- ใช้ API ที่มีรูปแบบสอดคล้องกัน พร้อม validation และข้อความผิดพลาดที่เข้าใจได้
- รหัสผ่านต้องถูก hash ก่อนบันทึก และ endpoint ที่ต้องยืนยันตัวตนต้องได้รับการป้องกัน
- ผู้ใช้ต้องเห็นสถานะการโหลด ความสำเร็จ และความผิดพลาดของการทำงานสำคัญ
- โครงสร้างโปรเจกต์ควรแยกความรับผิดชอบเป็น Web, API และ Mobile ตามโฟลเดอร์ `apps/`

## 7. สถาปัตยกรรมที่คาดหวัง

- **Web:** React สำหรับหน้าร้านและแดชบอร์ดผู้ดูแลระบบ
- **API:** Node.js และ Express สำหรับ REST API, authentication และ business logic
- **Database:** MongoDB สำหรับผู้ใช้ สินค้า ตะกร้า และคำสั่งซื้อ
- **Mobile:** พื้นที่สำหรับไคลเอนต์มือถือใน `apps/mobile` (อยู่นอกขอบเขต MVP เว้นแต่กำหนดเพิ่มเติม)

## 8. ข้อมูลหลัก

| --- | --- |
| User | name, email, passwordHash, role, addresses |
| Product | name, description, price, category, images, stock, status |
| Cart | userId, items, subtotal |
| Order | userId, items, totals, shippingAddress, paymentStatus, orderStatus |
| Category | name, slug, image |

## 9. ตัวชี้วัดความสำเร็จของ MVP

- ผู้ใช้ใหม่สามารถสมัครสมาชิก เลือกสินค้า และสร้างคำสั่งซื้อได้สำเร็จ
- ผู้ดูแลระบบสามารถเพิ่มสินค้าและเปลี่ยนสถานะคำสั่งซื้อได้
- ข้อมูลสต็อกและยอดรวมคำสั่งซื้อถูกต้องตามรายการสินค้า
- หน้าใช้งานหลักทำงานได้ดีบนหน้าจอมือถือและเดสก์ท็อป

## 10. สิ่งที่อยู่นอกขอบเขต MVP

- ระบบชำระเงินจริงและการเชื่อมต่อ payment gateway
- คูปอง คะแนนสะสม รีวิวสินค้า และรายการโปรด
- การจัดส่งแบบเรียลไทม์หรือเชื่อมต่อผู้ให้บริการขนส่ง
- หลายภาษา หลายสกุลเงิน และหลายร้านค้า
- แอปมือถือที่พร้อมเผยแพร่จริง

## 11. คำถามที่ต้องตัดสินใจก่อนเริ่มพัฒนา

1. ร้านจะขายสินค้าประเภทใด และมีข้อมูลสินค้าเฉพาะทางใดบ้าง
2. ต้องการรองรับการชำระเงินแบบใดในระยะถัดไป
3. สต็อกควรถูกตัดเมื่อสร้างคำสั่งซื้อ เมื่อชำระเงิน หรือเมื่อจัดส่ง
4. ต้องมีผู้ดูแลระบบกี่ระดับสิทธิ์
5. ต้องการให้แอปมือถืออยู่ในขอบเขตของเวอร์ชันแรกหรือไม่
